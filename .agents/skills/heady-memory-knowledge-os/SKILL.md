# Heady™ Memory Knowledge OS

## When to Use This Skill

Use this skill when the user asks for:

- persistent assistant memory
- anticipatory or proactive assistance
- repo-to-docs conversion
- documentation hubs for AI ingestion
- structured executive and technical briefings
- continuity across sessions, devices, or projects

## Instructions

1. Separate memory and knowledge into layers:
   - user profile and preferences
   - active projects and commitments
   - durable facts and exclusions
   - repo or document sources
   - synthesized briefing artifacts
2. Decide what should be remembered automatically versus only by explicit approval.
3. Organize raw sources into a clear knowledge stack:
   - executive summary
   - architecture and services
   - strategic implications
   - source appendix
4. Keep memory retrieval purposeful and relevant to the current task.
5. Keep documentation optimized for both humans and machine ingestion:
   - short sections
   - stable headings
   - direct links
   - minimal duplication
6. Define how anticipatory suggestions should work:
   - based on prior goals
   - bounded by relevance
   - easy to dismiss or correct
7. If the user has many repositories or notes, group by function rather than by time.
8. End with:
   - Memory Model
   - Knowledge Structure
   - Suggestion Policy
   - Highest-Value Missing Inputs

## Output Pattern

- User Context Layer
- Source Map
- Briefing Stack
- Retrieval Rules
- Suggestion Rules

## Example Prompts

- Build a persistent memory and briefing layer for my latent OS
- Turn these repos into a knowledge hub my assistant can use
- Design anticipatory memory without making the assistant intrusive

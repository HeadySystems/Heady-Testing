# Projected repos

The public repos now have a concrete generation path through `scripts/project-verticals.mjs`, plus a manifest package that maps each repo to a source app or service in the monorepo, which formalizes the projection model implied by the public repo READMEs ([headyme-core README](https://github.com/HeadyMe/headyme-core/blob/main/README.md), [headyos-core README](https://github.com/HeadyMe/headyos-core/blob/main/README.md), [headymcp-core README](https://github.com/HeadyMe/headymcp-core/blob/main/README.md), [headysystems-core README](https://github.com/HeadyMe/headysystems-core/blob/main/README.md), [headyapi-core README](https://github.com/HeadyMe/headyapi-core/blob/main/README.md), [headyconnection-core README](https://github.com/HeadyMe/headyconnection-core/blob/main/README.md), [headybuddy-core README](https://github.com/HeadyMe/headybuddy-core/blob/main/README.md)).

Each generated projection includes:

- README
- package.json
- PROJECTION_MANIFEST.json
- SOURCE_MAP.md

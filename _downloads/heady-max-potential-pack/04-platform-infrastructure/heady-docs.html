Quick Start Guide

Get up and running with Heady in 5 minutes

1

Sign Up for an Account

Visit headysystems.com/signup to create your free Heady account. You'll have instant access to the dashboard and all developer tools.

2

Get Your API Key

Once logged in, navigate to Settings → API Keys in the dashboard. Click Generate New Key and copy your API key to a secure location. You'll use this to authenticate all API requests.

3

Make Your First API Call

Let's test the Inference API with a simple cURL request:

cURL
curl -X POST https://api.headysystems.com/v1/inference \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain quantum computing in simple terms",
    "strategy": "best-quality",
    "max_tokens": 256
  }'

Expected Response:

JSON
{
  "id": "inf_1a2b3c4d5e6f",
  "object": "inference.result",
  "created": 1678886400,
  "model": "heady-v3-large",
  "usage": {
    "prompt_tokens": 8,
    "completion_tokens": 142,
    "total_tokens": 150
  },
  "choices": [
    {
      "text": "Quantum computing uses quantum bits (qubits) that can exist in multiple states simultaneously...",
      "finish_reason": "stop"
    }
  ]
}
Tip: Replace YOUR_API_KEY with your actual API key from the dashboard.
4

Try Multi-Model Routing

Heady automatically routes your requests to the best model based on your strategy. Try the lowest-cost strategy:

cURL
curl -X POST https://api.headysystems.com/v1/inference \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "What are the benefits of remote work?",
    "strategy": "lowest-cost",
    "max_tokens": 200
  }'

Available Strategies:

  • best-quality - Uses the most capable model (higher cost)
  • lowest-latency - Optimizes for speed (moderate cost)
  • lowest-cost - Most economical option
  • specific-model - Routes to a specific model ID
5

Connect HeadyBuddy Widget

Embed the Heady chat widget on your website in just two lines of code:

HTML
<script src="https://headybuddy.org/widget.js"></script>
<script>
  HeadyBuddy.init({
    apiKey: 'YOUR_API_KEY',
    position: 'bottom-right',
    theme: 'light'
  });
</script>

Your chat widget is now live! Customize colors, position, and behavior in the Widget Settings.

Authentication

Secure your API requests with Heady's authentication methods

API Key Authentication

The simplest way to authenticate with Heady is using your API key. Include it in the Authorization header of every request.

Header Format

HTTP Header
Authorization: Bearer sk_live_4eC39HqLyjWDarhtT...

Example Request

cURL
curl -X POST https://api.headysystems.com/v1/inference \
  -H "Authorization: Bearer sk_live_4eC39HqLyjWDarhtT..." \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Hello, world!"
  }'
Security: Never commit your API keys to version control. Use environment variables instead.

OAuth 2.0 Authentication

For third-party integrations, use OAuth 2.0 with the Authorization Code flow.

Authorization Code Flow

Step 1: Redirect the user to Heady's authorization endpoint:

URL
https://auth.headysystems.com/oauth/authorize?
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://yourapp.com/callback&
  response_type=code&
  scope=inference:read,chat:write&
  state=random_state_string

Step 2: User authorizes your app and gets redirected with an authorization code.

Step 3: Exchange the code for an access token:

cURL
curl -X POST https://auth.headysystems.com/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code&
       code=AUTH_CODE&
       client_id=YOUR_CLIENT_ID&
       client_secret=YOUR_CLIENT_SECRET&
       redirect_uri=https://yourapp.com/callback"

Response:

JSON
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "ref_1a2b3c4d5e6f...",
  "scope": "inference:read chat:write"
}

Token Lifecycle

Property Description Value
Access Token Expiry Time until access token expires 1 hour (3600 seconds)
Refresh Token Expiry Time until refresh token expires 30 days
Refresh Endpoint URL to refresh expired tokens POST /oauth/token
Revocation Immediately invalidate a token POST /oauth/revoke

Refreshing an Access Token

When your access token expires, use the refresh token to get a new one:

cURL
curl -X POST https://auth.headysystems.com/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=refresh_token&
       refresh_token=ref_1a2b3c4d5e6f...&
       client_id=YOUR_CLIENT_ID&
       client_secret=YOUR_CLIENT_SECRET"

Revoking a Token

Immediately invalidate a token:

cURL
curl -X POST https://auth.headysystems.com/oauth/revoke \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "token=YOUR_TOKEN&
       client_id=YOUR_CLIENT_ID&
       client_secret=YOUR_CLIENT_SECRET"

Rate Limits

API rate limits vary by authentication method and tier:

Tier Requests/Minute Monthly Limit Concurrent Requests
Free 10 10,000 1
Pro 100 1,000,000 10
Enterprise 1,000 Unlimited 100
Rate Limit Headers: Check your remaining quota using the X-RateLimit-Remaining header in all API responses.

Error Codes

When authentication fails, you'll receive one of these error codes:

Code HTTP Status Description Solution
invalid_api_key 401 API key is missing or invalid Check your API key and try again
expired_token 401 OAuth token has expired Refresh the token using your refresh token
insufficient_scope 403 Token lacks required permissions Request new auth with required scopes
rate_limit_exceeded 429 Too many requests in short time Wait before retrying or upgrade plan
revoked_token 401 Token has been revoked Authenticate again with new credentials
invalid_grant 400 Authorization code or refresh token invalid Restart OAuth flow or check token validity

Inference Endpoint

Generate text completions with the Heady inference engine

Endpoint

HTTP
POST /v1/inference

Request Body

Parameter Type Required Description
prompt string Yes The input text to generate from (max 8,000 tokens)
strategy string No Routing strategy: best-quality, lowest-latency, lowest-cost, specific-model (default: best-quality)
model string No Model ID when using specific-model strategy
max_tokens integer No Maximum tokens to generate (1-4096, default: 256)
temperature float No Sampling temperature (0.0-2.0, default: 0.7)
top_p float No Nucleus sampling parameter (0.0-1.0, default: 1.0)
stop_sequences array No Strings that stop generation when encountered
stream boolean No Stream response tokens as they're generated (default: false)

Response Schema

JSON
{
  "id": "string",                    // Unique inference ID
  "object": "inference.result",
  "created": number,                 // Unix timestamp
  "model": "string",                 // Model used for generation
  "usage": {
    "prompt_tokens": number,
    "completion_tokens": number,
    "total_tokens": number
  },
  "choices": [
    {
      "text": "string",               // Generated text
      "finish_reason": "stop",       // stop, length, or error
      "index": 0
    }
  ]
}

Strategy Options

best-quality

Selects the most capable model available. Best for complex tasks, reasoning, and creative work. Higher latency and cost.

lowest-latency

Optimizes for response speed with good quality. Ideal for real-time applications and chat interfaces.

lowest-cost

Uses the most economical model that meets quality thresholds. Best for high-volume applications.

specific-model

Routes to a specific model by ID. Requires the model parameter.

Code Examples

cURL

cURL
curl -X POST https://api.headysystems.com/v1/inference \
  -H "Authorization: Bearer sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Write a haiku about programming",
    "strategy": "best-quality",
    "max_tokens": 100,
    "temperature": 0.8
  }'

JavaScript

JavaScript
const response = await fetch(
  'https://api.headysystems.com/v1/inference',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer sk_live_...',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      prompt: 'What are the benefits of TypeScript?',
      strategy: 'best-quality',
      max_tokens: 256,
      temperature: 0.7
    })
  }
);

const data = await response.json();
console.log(data.choices[0].text);

Python

Python
import requests

response = requests.post(
  'https://api.headysystems.com/v1/inference',
  headers={
    'Authorization': 'Bearer sk_live_...',
    'Content-Type': 'application/json'
  },
  json={
    'prompt': 'Explain machine learning to a beginner',
    'strategy': 'best-quality',
    'max_tokens': 256,
    'temperature': 0.7
  }
)

data = response.json()
print(data['choices'][0]['text'])

500 internal_error Server error during inference
503 service_unavailable Heady service is temporarily unavailable

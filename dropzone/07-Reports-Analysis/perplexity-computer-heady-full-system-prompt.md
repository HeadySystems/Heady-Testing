## 🔐 UNIFIED FIREBASE AUTHENTICATION

### Architecture

```
Firebase Auth (Google OAuth + Email/Password + Anonymous)
    → Central auth domain: auth.headysystems.com
    → Relay iframe syncs tokens to all 9 site domains via postMessage
    → Store session in Firebase Firestore (user profile, preferences)
    → Sync to PostgreSQL/pgvector (vector memory persistence)
    → HeadyAutoContext indexes user context into 3D vector memory
    → ALL sites reflect auth state + personalized content
```

### Central Auth Domain (`auth.headysystems.com`)

Firebase Auth is domain-scoped. A **central auth domain** is required for cross-site auth:

- Host the auth page at `auth.headysystems.com/login`
- All 9 sites redirect here for sign-in with `?redirect=ORIGINATING_URL`
- On sign-in success, mint a **custom Firebase token** server-side
- Relay token to all domains via **hidden relay iframe + postMessage**
- Store tokens as **httpOnly, Secure, SameSite=Strict cookies** — NOT localStorage (XSS vulnerable)
- Validate redirect URLs against server-side allowlist (CSRF protection)
- Use **state/nonce params** in all OAuth flows

### Auth Page Design

- Premium dark glassmorphism on `auth.headysystems.com`
- Google OAuth, Email/Password, Anonymous guest
- `heady_user = { uid, email, displayName, photoURL, provider }`
- Short JWT expiry (15-60 min) + refresh tokens
- Revocation blacklist in Firestore for logout

### Auth Widget (embed in ALL sites)

- Float in top-right nav
- Show avatar when signed in, "Sign In" when not
- Auto token refresh via relay iframe heartbeat
- Emit `heady:auth:changed` custom events for reactivity
- Handle sign-out globally across all sites via central domain
- **Firestore security rules:** `allow read,write: if request.auth.uid == resource.data.uid`
- Rate-limit anonymous sign-ins to prevent abuse

What is HeadyOS

Liquid · Latent · Living

HeadyOS is more than an orchestration platform — it's a living system. Every component breathes, self-corrects, and adapts to your workload in real time, guided by the HCFullPipeline engine and Sacred Geometry design principles.

From a single developer to distributed AI agent swarms, HeadyOS provides the intelligence layer that ties together containers, models, data pipelines, and human workflows into one fluid, observable experience.

⚡

Intelligent Orchestration

HCFullPipeline runs stages in strict dependency order with deep self-analysis at every checkpoint. The system detects drift, heals errors, and optimises automatically.

🧠

Multi-Agent Supervisor

Parallel fan-out routing dispatches tasks to specialised agents (Builder, Researcher, Claude Code, Deployer) and aggregates results deterministically.

∞

Sacred Geometry UX

Every interface follows the Sacred Geometry design language — organic rounding, breathing animations, and a gold/cyan/magenta palette that signals system state visually.

HCFullPipeline

Five Stages. One Truth.

01

Ingest

Collects data, configs, and task definitions from all registered sources. Validates schemas and performs initial integrity checks.

02

Plan

Builds the stage dependency graph. Evaluates resource policies, cost budgets, and concurrency limits before committing any execution.

03

Execute — Major Phase

Parallel fan-out to agents. Builder, Researcher, Claude Code, and Deployer run concurrently, supervised by HeadyBrain's multi-agent router.

04

Recover

Detects failed tasks and applies circuit-breaker patterns. Attempts automated repair before escalating to the owner. ORS score drives recovery intensity.

05

Finalize

Aggregates results, updates the HeadyRegistry, syncs documentation, and emits a comprehensive checkpoint report.

Built for Determinism

Given identical inputs — the same hcfullpipeline.yaml, resource-policies.yaml, and dependency versions — HeadyOS always produces the same execution plan and routing decisions. Randomness is seeded and logged per run, making every result reproducible.

The Operational Readiness Score (ORS) gates how aggressively the system executes. Above 85 it runs at full parallelism. Below 50 it enters recovery-only mode and halts new builds until health is restored.

Quick Start

Up in 3 Steps

From zero to a running HeadyOS node in under five minutes.

  1. 1

    Clone the repository

    Pull the HeadyOS monorepo from GitHub. All services, configs, and frontend assets are included.

  2. 2

    Configure environment

    Copy .env.example to .env and fill in your API keys (Anthropic, Render, Stripe). All other defaults are pre-configured.

  3. 3

    Start the manager

    Run npm start. The manager boots on port 3300, loads all YAML configs, and registers every subsystem. Open your browser and navigate here.

SDK & Integration Patterns

Using the HeadyOS SDK to build integrations

The HeadyOS SDK provides a comprehensive set of tools and patterns for integrating external systems with the platform. Built on TypeScript, it offers full type safety and IntelliSense support.

Getting the SDK
bash 
npm install @headyos/sdk

Basic Integration Example

typescript 
import { HeadyOS } from '@headyos/sdk';

const heady = new HeadyOS({
  apiKey: process.env.HEADY_API_KEY,
  endpoint: 'https://api.headyos.io'
});

// Register a custom service
await heady.services.register({
  id: 'my-service',
  name: 'My Custom Service',
  capabilities: ['process', 'analyze', 'report']
});

// Execute a pipeline
const result = await heady.pipelines.execute('hc-full-pipeline', {
  data: { input: 'value' }
});

console.log('Pipeline result:', result);

MCP Tool Development

Create Model Context Protocol tools for AI integration

MCP (Model Context Protocol) tools enable AI models to interact with external systems. Develop tools to expose your service functionality to Claude and other AI systems.

typescript 
import { MCPServer, Tool } from '@headyos/mcp-sdk';

const server = new MCPServer({
  name: 'my-mcp-server',
  version: '1.0.0'
});

// Define a tool
const analyzeDataTool: Tool = {
  name: 'analyze_data',
  description: 'Analyze data and return insights',
  inputSchema: {
    type: 'object',
    properties: {
      data: { type: 'array', description: 'Data to analyze' },
      method: { type: 'string', enum: ['statistical', 'ml', 'rule-based'] }
    },
    required: ['data', 'method']
  },
  execute: async (input) => {
    // Implement tool logic
    const results = await analyzeDataLogic(input.data, input.method);
    return { insights: results };
  }
};

server.registerTool(analyzeDataTool);
await server.start();

Custom Pipeline Stages

Create and integrate custom transformation stages

Extend the HC Full Pipeline with custom stages for specialized data processing, transformation, and business logic.

typescript 
import { PipelineStage, StageContext } from '@headyos/sdk';

export class CustomTransformStage implements PipelineStage {
  name = 'custom-transform';
  description = 'Apply custom business logic transformations';

  async execute(context: StageContext) {
    const { input, config } = context;
    
    // Custom transformation logic
    const transformed = this.applyTransformation(input, config);
    
    // Emit telemetry
    await context.telemetry.record({
      stage: this.name,
      inputSize: input.length,
      outputSize: transformed.length,
      duration: Date.now() - context.startTime
    });

    return { output: transformed };
  }

  private applyTransformation(input: any[], config: any) {
    // Implementation
    return input.map(item => ({
      ...item,
      processed: true,
      timestamp: new Date().toISOString()
    }));
  }
}

Stage Lifecycle

  • Initialize: Set up resources and configuration
  • Execute: Process input data through the stage
  • Validate: Verify output meets requirements
  • Cleanup: Release resources and record metrics

HeadyBee Worker Development

Create autonomous worker agents for background tasks

HeadyBee workers are autonomous agents that perform background tasks, handle event processing, and manage long-running operations within the HeadyOS ecosystem.

typescript 
import { HeadyBeeWorker } from '@headyos/heady-bee';

class MyWorker extends HeadyBeeWorker {
  name = 'my-background-worker';

  async onStart() {
    this.logger.info('Worker started');
    await this.subscribeToEvents('data:incoming');
  }

  async onEvent(event: any) {
    try {
      // Process event
      await this.processData(event.data);
      
      // Emit result event
      await this.emit('data:processed', { 
        originalId: event.id,
        status: 'success'
      });
    } catch (error) {
      await this.emit('data:error', { 
        originalId: event.id,
        error: error.message
      });
    }
  }

  private async processData(data: any) {
    // Business logic
    return await this.analyze(data);
  }
}

const worker = new MyWorker();
await worker.start();

Liquid Node Creation

Bridge protocols and systems with protocol adapters

Liquid Nodes are protocol bridge nodes that enable HeadyOS to communicate with external systems using different protocols (MCP, HTTP, WebSocket, UDP, MIDI, TCP).

typescript 
import { LiquidNode, ProtocolAdapter } from '@headyos/liquid-nodes';

// Create a custom protocol adapter
class MyCustomAdapter implements ProtocolAdapter {
  protocol = 'custom';

  async encode(message: any): Promise {
    // Convert message to protocol format
    return Buffer.from(JSON.stringify(message));
  }

  async decode(data: Buffer): Promise {
    // Parse protocol format to message
    return JSON.parse(data.toString());
  }
}

// Register adapter and create Liquid Node
const node = new LiquidNode();
node.registerAdapter(new MyCustomAdapter());

await node.bridge({
  source: { protocol: 'http', endpoint: 'http://my-service.local' },
  target: { protocol: 'custom', endpoint: 'custom://downstream' }
});

Supported Protocols

  • MCP (Model Context Protocol)
  • HTTP/HTTPS (REST, GraphQL)
  • WebSocket (bidirectional streaming)
  • UDP (lightweight messaging)
  • MIDI (music and device control)
  • TCP (custom protocols)

Authentication & API Keys

Secure your integrations with proper authentication

HeadyOS supports multiple authentication methods to secure your integrations and API communications.

API Key Authentication

typescript 
const heady = new HeadyOS({
  apiKey: process.env.HEADY_API_KEY,
  apiSecret: process.env.HEADY_API_SECRET

  provider: 'github',

  clientSecret: process.env.GITHUB_CLIENT_SECRET,

Security Guidelines

Never commit API keys to version control. Use environment variables or secret management systems. Rotate keys regularly and revoke unused credentials.

Example: Resilient Integration

typescript 
async function executeWithRetry(
  fn: () => Promise,
  maxRetries: number = 3,
  backoffMs: number = 100
) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await new Promise(r => setTimeout(r, backoffMs * Math.pow(2, i)));
    }
  }
}

// Usage
const result = await executeWithRetry(() => 
  heady.pipelines.execute('pipeline-name', data)
);

GitHub Integration

Listen to GitHub events, trigger pipelines on push/PR, post updates via HeadyOS.

typescript 
const github = await heady.integrations.github.connect({
  token: process.env.GITHUB_TOKEN
});

github.on('push', async (event) => {
  await heady.pipelines.execute('ci-pipeline', {
    repo: event.repository.full_name,
    branch: event.ref,
    commit: event.head_commit.id
  });
});

Database Integration

Query databases, stream results, trigger actions based on data changes.

typescript 
const db = await heady.integrations.database.connect({
  type: 'postgresql',
  url: process.env.DATABASE_URL
});

const results = await db.query('SELECT * FROM events WHERE status = $1', ['pending']);

for (const result of results) {
  await heady.pipelines.execute('event-processor', { event: result });
}

Liquid Nodes

MCP-Connected External Interfaces for HeadyOS

Liquid Nodes are dynamic connectors that extend HeadyOS capabilities through the Model Context Protocol (MCP). Each node bridges Claude, HeadyManager, and external services—from GitHub repositories to Cloudflare infrastructure to AI inference endpoints—with unified token-based authentication.

Node Connection Setup Guide

Step-by-step instructions for connecting each Liquid Node to HeadyOS.

1
GitHub Node (SCM)

Generate a personal access token with repo, gist, and read:org scopes.

export GITHUB_TOKEN="ghp_your_token_here"

Verify: heady liquid-nodes verify github

2
Cloudflare Node (Infrastructure)

Create an API token in Cloudflare dashboard with Zone.Zone:Read and DNS:Edit permissions.

export CLOUDFLARE_API_TOKEN="v1.0_your_token_here"

Verify: heady liquid-nodes verify cloudflare

3
Vertex AI Node (ML)

Set up GCP service account key and configure project ID.

export GCP_PROJECT_ID="your-project-id" export GOOGLE_APPLICATION_CREDENTIALS="/path/to/service-account.json"

Verify: heady liquid-nodes verify vertex-ai

4
AI Studio Node (LLM)

Create API key in Google AI Studio console (makersuite.google.com).

export GOOGLE_AI_API_KEY="AIzaSyD_your_key_here"

Verify: heady liquid-nodes verify ai-studio

5
Colab Node (Notebooks)

Authenticate with Google Drive via OAuth flow.

heady liquid-nodes auth google-drive

Verify: heady liquid-nodes verify colab

6
Restart HeadyManager

Apply configuration changes and activate all connected nodes.

heady manager restart

Monitor: heady liquid-nodes status

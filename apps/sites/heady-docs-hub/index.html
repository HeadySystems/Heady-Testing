Evolution Engine

Evolutionary optimization engine that improves system configurations through phi-scaled mutation, CSL-gated selection, and tournament-based population management.

Key Features

  • Population size: fib(8) = 21 candidates
  • Mutation rate: ψ³ ≈ 0.236 (phi-derived, not arbitrary)
  • Crossover probability: ψ ≈ 0.618
  • Tournament size: fib(5) = 5
  • Elite preservation: top fib(5) = 5 candidates pass unchanged
  • CSL-gated fitness: candidates must pass MEDIUM threshold (0.809)

Voting Mechanism

Responses are embedded and clustered by cosine similarity. The largest cluster gets the highest weight (ψ0), second cluster gets ψ1, and so on — phi-fusion weights that decay geometrically. Dissenting responses (outside all clusters) are flagged for manual review if their CSL gate score exceeds the MEDIUM threshold.

Key Functions

shared/phi-math.js
// CSL Threshold Generator function phiThreshold(level, spread = 0.5) { return 1 - Math.pow(PSI, level) * spread; } // MINIMUM: 0.500, LOW: 0.691, MEDIUM: 0.809, HIGH: 0.882, CRITICAL: 0.927 // Phi-Weighted Fusion (geometric decay) function phiFusionWeights(n) { const raw = Array.from({ length: n }, (_, i) => Math.pow(PSI, i)); const sum = raw.reduce((a, b) => a + b, 0); return raw.map(w => w / sum); } // Phi-Backoff (exponential with golden ratio base) function phiBackoff(attempt, baseMs = 1000, maxMs = 60000) { return Math.min(Math.round(baseMs * Math.pow(PHI, attempt)), maxMs); } // CSL Gate (soft sigmoid threshold) function cslGate(value, cosScore, tau, temp = 0.236) { return value * (1 / (1 + Math.exp(-(cosScore - tau) / temp))); }

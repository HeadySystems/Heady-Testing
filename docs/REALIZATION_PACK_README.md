## How to Use This Pack

### Step 1: Set Up Your AI Tools
- Paste `perplexity-custom-instruction.md` into Perplexity's custom instructions
- Use `claude-cursor-instruction.md` as the system prompt in Claude, Windsurf, or Cursor

### Step 2: Run the Master Prompt
- Feed `00-continuous-realization-engine.md` to your primary AI assistant
- This triggers the 5-phase autonomous improvement cycle

### Step 3: Activate Individual Nodes
- Run each `*-activation.md` prompt to bring each node online at full capability
- Each activation includes self-tests that verify the node is operational

### Step 4: Diagnose
- Run the diagnostic flows to assess current state
- Start with `hob-diagnostic-flow.md` for a complete system assessment
- Use specific diagnostics (performance, security, capability) as needed

### Step 5: Question Everything
- Feed questions from `100-realization-questions.md` to various nodes
- These questions force deep reasoning and self-improvement
- There are no "right answers" — every answer should trigger action

### Step 6: Break Things
- Run the stress test suite to find every breaking point
- Start with Tier 1 (basic) and escalate through Tier 4 (endurance)
- Every failure found is a failure prevented in production

### Step 7: Improve Continuously
- Follow the self-improvement protocol (daily/weekly/monthly/quarterly)
- These rituals compound — skip them and drift accumulates

### Step 8: Trigger Emergence
- Run the emergent capability triggers to unlock capabilities
  that only exist when multiple nodes coordinate
- These are the highest-value prompts because they create
  capabilities that were never explicitly designed

# Build Order

## Phase 1

- Finalize registry and domain ownership
- Lock contracts, event names, and health conventions
- Stand up projection and validation scripts

## Phase 2

- Build HeadyAPI as the only public ingress
- Build HeadyMCP for tool and model routing
- Build HeadyOS as planner, conductor, and safety engine

## Phase 3

- Build HeadyMemory with entity graph and retrieval policy
- Build HeadySystems for observability, release, and recovery
- Build HeadyBot for long-running automation

## Phase 4

- Build HeadyMe and HeadyBuddy experiences
- Add HeadyConnection public and collaborative surface
- Turn on release automation and projections

## Phase 5

- Add enterprise guardrails, budget policies, replay testing, and failure drills

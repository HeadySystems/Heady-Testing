# Architecture Overview

## Goal

Rebuild Heady as a latent operating system with clean ownership boundaries, predictable release flows, and a projection model that keeps public repos aligned with one private or primary source monorepo.

## Core principle

The monorepo is the operating truth.
Projected repos are delivery surfaces.

## Layers

### Experience layer

- HeadyMe: command center for the user
- HeadyBuddy: companion and creative workspace
- HeadyConnection: mission, collaboration, and nonprofit-facing layer

### Intelligence layer

- HeadyAPI: external access, auth, routing, quotas
- HeadyMCP: tool and model context protocol gateway
- HeadyOS: planner, conductor, orchestration, runtime policy
- HeadyMemory: memory, vector context, entity graph, retrieval
- HeadyBot: automation and autonomous task execution

### Infrastructure layer

- HeadySystems: deploy, observe, secure, recover, and project

## Why this is stronger

- Shared contracts live once
- Domain mapping is centralized
- Projection is automated instead of manually copied
- Each core repo becomes a controlled view of the same ecosystem

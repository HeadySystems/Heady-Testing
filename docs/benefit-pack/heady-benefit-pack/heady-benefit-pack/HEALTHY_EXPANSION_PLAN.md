# Heady Expansion Plan

## Core platform additions
- Permission graph with scoped token delegation and action audit receipts
- Temporal and privacy-aware memory model with user, session, and agent scopes
- Unified service manifest for liquid app and connector delivery
- Edge-delivered WASM module strategy for low-latency UI/runtime surfaces
- Canonical environment contract and secret hygiene enforcement
- Archive/supersede registry for stale repos, mirrors, and legacy experiments
- Cross-device identity continuity with device trust and session handoff
- Manager surface for asynchronous orchestration across workstreams
- Knowledge base that persists implementation artifacts, decisions, and lessons
- MCP-first connector model for tools, memory, browser actions, and IDE workflows

## HeadyWeb additions
- Browser-shell launch surface for all Heady experiences
- Side-panel Buddy, IDE, and command-center embedding
- Work-area/tab-group isolation concept
- Command palette for navigation, actions, and orchestration
- PWA installability with offline shell and fast resume
- Accessibility-first keyboard and reduced-motion support
- Browser-memory layer for cross-tab and cross-session continuity
- Lightweight local footprint with cloud-heavy execution routing

## HeadyBuddy additions
- Observer / Actor / Autonomous permission tiers
- Explicit action preview and approval flow for writes
- Android work-profile owner mode plan
- Windows and Linux isolated work-area packaging strategy
- Cross-device sync, handoff, and notification continuity
- Persistent personal memory with privacy controls and temporal validity
- Site interaction packs and skill-style capability extensions
- Sandboxed automation sessions separate from personal browsing sessions

## HeadyAI-IDE additions
- Editor + Manager dual-surface model
- Project knowledge base and artifact memory
- Cloud execution with thin local shell
- Browser-integrated testing and preview through HeadyWeb
- Command palette, activity bar, status bar, and AI action rail
- Multi-model routing and arena-style comparison surface
- MCP host behavior for extensibility
- Browser-delivered installable package strategy

## Latent OS additions
- Liquid projection registry for apps, UIs, connectors, and workflows
- Context-sensitive delivery of the right surface at the right time
- Cell-based routing for performance and blast-radius reduction
- Memory MCP service for universal contextual recall
- Unified orchestration receipts and system trace visibility
- App/module manifest for streaming delivery and recovery

Overview

Introduction to HeadyOS

Welcome to the official documentation for HeadyOS. This platform serves as the os interface — interactive headylatentos management and monitoring. It is a core component of the broader Heady ecosystem, built on principles of sacred geometry, organic systems, and breathing interfaces.

Our infrastructure is designed to provide unparalleled performance, security, and scalability. By deeply integrating with the Heady Model Context Protocol (MCP) and the HCFullPipeline, HeadyOS ensures seamless operation across all layers of the stack.

High Performance

Built on top of Cloudflare edge network and Google Kubernetes Engine for sub-millisecond latency.

Brain-Aware

Fully integrated with the HeadyBrain orchestrator for intelligent, context-aware routing.

Zero Trust Security

Continuous authentication and Continuous Security Posture Management (CSPM) enforced at every node.

Architecture

System Architecture

HeadyOS utilizes a microservices-based architecture deployed across multiple global regions. The system is conceptually divided into several key layers:

  • Edge Layer: Handles global routing, DDoS protection, and TLS termination via Cloudflare.
  • Gateway Layer: API Gateway that validates requests, enforces rate limits, and performs JWT verification.
  • Orchestration Layer: The HCSysOrchestrator manages the flow of data between services based on cognitive load and system health.
  • Service Layer: The core business logic specific to headyos.com.
  • Data Layer: Distributed Postgres databases with real-time replication and vector stores for AI embeddings.

const topology = {
  edge: 'cloudflare-workers',
  gateway: 'envoy-proxy',
  orchestrator: 'hcsys-orchestrator',
  compute: 'gke-clusters',
  storage: ['postgres', 'redis-cluster', 'qdrant-vector']
};
Integration

API Reference

Interact with HeadyOS programmatically using our REST and MCP APIs. All endpoints require authentication via a standard Bearer token.

Authentication

Include your API key in the Authorization header of your requests:

curl -H "Authorization: Bearer hdy_your_api_key" https://api.headyos.com/v1/status

Common Endpoints

Endpoint Method Description
/v1/health GET Returns the current health status of the service cluster.
/v1/metrics GET Provides Prometheus-formatted metrics for monitoring.
/v1/sync POST Triggers a synchronization event with the Heady Registry.
/mcp/connect WS Establishes a WebSocket connection for real-time MCP streaming.

OS Architecture

Liquid Latent Operating System

HeadyOS is not a traditional operating system. It is a Liquid Latent OS where all components map into continuous 3D vector space with 384D embeddings. The icosahedron's 12 vertices represent the core subsystems, its 20 faces the interaction surfaces.

VEC

Vector-Space Routing

Every system call, every process, every resource allocation is represented as a vector in 384D space. Routing decisions use cosine similarity to find the optimal path through the system topology.

PHI

Phi-Derived Thresholds

All system thresholds derive from the golden ratio: CPU allocation pools at Hot=34%, Warm=21%, Cold=13%, Reserve=8%, Governance=5%. Backoff uses PHI^attempt with 38.2% jitter.

CSL

CSL Gate System

Continuous Semantic Logic gates every operation. Confidence thresholds from 0.500 (MINIMUM) to 0.927 (CRITICAL) determine which operations proceed, ensuring semantic coherence across all layers.

ORC

Conductor-Orchestrator

The HeadyConductor manages task routing and node selection across the sacred geometry topology. From HeadySoul at center through Inner, Middle, and Outer rings to the Governance layer.

System Services

175 Orchestrated Services

HeadyOS runs 175 registered services, each with its own /health endpoint returning status, coherenceScore, and version. Services communicate through the sacred geometry topology with CSL-gated routing.

GW

Gateway Service

The central API Gateway running on port 3330. Handles CSL-routed requests with phi-scaled rate limiting. All external traffic enters through this service before being distributed to internal nodes.

MGR

Manager Service

The MCP server and API gateway on port 3301. Manages MCP protocol connections, routes requests to brain nodes, and maintains the service registry for all 175 registered services.

BRN

Brain Service

The cognitive processing center on port 3302. Handles AI inference, embedding generation, and semantic reasoning. Connected to vector stores and the latent space tunnels at vector.headyos.com.

CND

Conductor Service

Task routing and node selection on port 3303. The TypeScript-based conductor evaluates CSL confidence scores to route tasks to the optimal node in the sacred geometry topology.

PRS

Persistence Service

User state persistence on port 3340. Implements CRDT synchronization for conflict-free distributed state management across sessions with phi-proportioned garbage collection intervals.

BDY

Buddy Service

The HeadyBuddy backend on port 3310. Powers the AI companion interface across web widget, Android app (island-style work profile), and Tauri desktop application.

Integration

API Reference

Interact with HeadyMe programmatically using our REST and MCP APIs. All endpoints require authentication via a standard Bearer token.

Authentication

Include your API key in the Authorization header of your requests:

curl -H "Authorization: Bearer hdy_your_api_key" https://api.headyme.com/v1/status

Common Endpoints

Endpoint Method Description
/v1/health GET Returns the current health status of the service cluster.
/v1/metrics GET Provides Prometheus-formatted metrics for monitoring.
/v1/sync POST Triggers a synchronization event with the Heady Registry.
/mcp/connect WS Establishes a WebSocket connection for real-time MCP streaming.

Trust & Safety

Your Data,
Your Sovereignty

Privacy isn't a feature -- it's the foundation. HeadyMe is built with zero-trust architecture and end-to-end encryption from day one.

End-to-End Encryption

All data encrypted at rest and in transit. Your conversations and memory vectors are protected with AES-256-GCM encryption.

Zero-Knowledge Architecture

We cannot read your data. Period. Your encryption keys are derived locally and never transmitted to our servers.

Full Transparency

Open audit logs, clear data policies, and granular controls. Know exactly what data exists and delete it anytime with one click.

Local-First Processing

Sensitive operations run on-device when possible. Your most private data never leaves your hardware unless you explicitly allow it.
